Jaiminiya Samavedam

This project converts the Sama Vedam text present in the PDF file S1.pdf into Unicode text. The document was created using a proprietary font called Krishna Vedic. Below are the steps followed to convert the document:

  1. Extract the embedded fonts from the Adobe PDF document using an online tool. These fonts are stored in the fonts directory. The majority of the texts use the Krishna Vedic font.
  2. Create a set of unique ASCII characters present in the PDF file and print the characters using that font.
  3. Use a tool like FontBook (on Mac) to find the corresponding Unicode values.
  4. Create the mapping in the file char_map.py.
  5. Handle certain idiosyncrasies (e.g., some characters need to be reversed, or require a different ordering).
  6. The mapping is present in the file output_text/mapping.html.
  7. The generated Grantha file is available at output_text/pages-Grantha/render-finaljson.html.
  8. The generated Devanagari file is available at output_text/pages-Devanagari/render-finaljson.html.
  9. The generated Tamil file is available at output_text/pages-Tamil/render-finaljson.html.
  10. The generated Malayalam file is available at output_text/pages-Malayalam/render-finaljson.html.
  11. This is still work in progress and will get updated regularly.
  12. This can be converted into other Unicode languages like Devanagari using converters such as Aksharamukha.

Correction Process

  1. We are using the Devanagari file for correction rather than Grantha
  2. Once the proof reading is complete the text will be transliterated into Grantha, Malayalam and Tamil
  3. The json file is treated as the sole source of truth. The correction process is the following workflow
    1. Get the open issues from Github using the script get_issues.py
    2. Convert the issues into json and create a updated_Devanagari.json file
    3. Convert the updated_Devanagari.json file to .tex file
    4. Convert the .tex to PDF using LuaLatex
    5. Check in the PDF to Github
  4. This is the workflow for the correction process if the text file is used :
    1. The text file needs to be of the specific format . -- A line starting with the # and another line ending with the # and corrections in between
    2. There can be multiple corrections in a file
    3. The files can either be checked in using a Pull request [GitHub PR mechanism](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request)
    4. If this is not possible we can explore alternative means
    5. The end result is to obtain a [corrected json file](output_text/corrected-Devanagari.json) that incorporates all changes.
    6. The corrected json file is then merged back into the main json file using the [script](apply-corrections.py)
  5. There are a couple of challenges in the above workflow:
    1. The headings are not editable
    2. The expectation is that the number of lines should be the same in the corrected version. This is not ideal and results in one more correction cycle to just fix the sentences. So for now the correction cycle fixes the swaras and the mantras etc. not the number of sentences. This has now been fixed
  6. There is a new text file added from Release 0.8 which can be used to edit headings etc. Changes made to this file can be checked in via the [GitHub PR mechanism](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request).
  7. This is the workflow to be followed. This can be run simulatenously with the other workflow to get issues from the GitHub
    1. Create a file in the directory output_text/txt/corrections . The file name has to be in the format corrections_nnn.txt
    2. This file should contain the section name marker, and one or more subsection markers
    3. You can have multiple sections in.a file. Each section can contain one more subsections
    4. Run the command python3 get_text_corrections.py This will process all the files present and produce one output file output_text/corrected-Devanagari-from-txt.json
    5. The corrected json file is then merged back into the main json file using the apply-corrections_text

How to Run the Code

  1. Clone or download the repository.
  2. Create a Python environment and install dependencies: 
    pip install -r requirements.txt
  3. Where to modify: If you find mistakes in the character mapping, edit the char_map.py file. The mapping can be viewed in output_text/mapping.html.
  4. How to generate the text file: 
    python3 extract_pdf_text.py
    This will generate the output file. output_text/output_grantha.txt . This file has some errors . The errors seem to be inserted during the extraction. These errors needs to be fixed in the output put file before proceeding to the next stage. This contains the code for mapping the propreitary characters to unicode. 
  5. python3 converttojson.py output_text/output_grantha.txt > converttojson_output.txt
    This will generate the json file. output_text/final-Grantha.json 
  6. python groupByWords.py > groupByWords_output.txt
    This will generate the json file. output_text/final-Grantha.json 
  7. python3 test-read.py
    This will generate the json file. output_text/rewritten_final-Grantha.json

    This will generate the json files.
